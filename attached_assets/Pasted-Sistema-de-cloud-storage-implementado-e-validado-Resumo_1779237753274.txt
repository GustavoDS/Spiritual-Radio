Cloud Storage — Implementado
Arquitetura de providers
Três providers com interface comum StorageProvider:

Provider	Env var	Quando usar
local	STORAGE_PROVIDER=local (padrão)	Desenvolvimento
s3	STORAGE_PROVIDER=s3	AWS S3
r2	STORAGE_PROVIDER=r2	Cloudflare R2 — recomendado produção
Interface completa (8 métodos):

upload(localPath, key, options?)  → Promise<string>  // URL pública
delete(key)                        → Promise<void>
exists(key)                        → Promise<boolean>
getUrl(key)                        → string
getSignedUrl(key, expiresIn?)      → Promise<string>
move(fromKey, toKey)               → Promise<string>
copy(fromKey, toKey)               → Promise<string>
listFiles(subdir)                  → Promise<string[]>

R2StorageProvider (novo)
Endpoint automático: https://${R2_ACCOUNT_ID}.r2.cloudflarestorage.com
Usa @aws-sdk/client-s3 + @aws-sdk/lib-storage (multipart streaming)
Upload: envia arquivo → deleta temp local → retorna URL CDN pública
listFiles() com paginação via ListObjectsV2Command
move() / copy() via CopyObjectCommand
MIME type detectado automaticamente pela extensão
Upload flow corrigido em contents.controller.ts
Antes (local disk direto, bypassa cloud):

audio_url: files?.["audio"]?.[0]?.path
  ? filePathToUrl(files["audio"][0].path)  // ← nunca chegava ao S3/R2

Agora (passa pelo provider):

await storageProvider.upload(file.path, buildStorageKey("audio", file.originalname))
// local → /uploads/audio/filename.mp3 (no-op, já estava em disco)
// s3/r2 → https://cdn.domain.com/audio/filename.mp3 (upload real)
