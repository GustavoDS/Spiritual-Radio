Vinhetas com bed musical + SFX
Contexto
As vinhetas atualmente são apenas TTS "seco" (voz isolada), o que destoa de uma rádio profissional. Precisamos enriquecer com:

SFX stinger curto antes e depois da voz (whoosh / chime / sweep)
Bed musical em loop por baixo da voz, com volume reduzido e ducking opcional
Normalização de loudness (-16 LUFS) para nivelar com o resto do conteúdo
Cache no R2 dos assets gerados (SFX) e do áudio final mixado
A mixagem é feita com ffmpeg (já disponível no servidor). SFX é gerado via ElevenLabs Sound Effects API (POST https://api.elevenlabs.io/v1/sound-generation).

3. Pipeline de mixagem — atualizar vinhetas.service.ts → gerarAudio(id, voiceId?)
Substituir o fluxo atual (TTS direto) por:

1. Gerar TTS da voz   → /tmp/{id}_voice.mp3
2. Resolver assets:
   - sfx_intro_url  (na vinheta; se faltar, busca em SFX_CATALOG por tipo)
   - sfx_outro_url  (idem; opcional)
   - bed_url        (background_track.audio_url, se background_track_id)
3. Baixar tudo para /tmp
4. ffmpeg mix (ver comando abaixo)
5. Loudness normalize (loudnorm I=-16 LRA=11 TP=-1.5)
6. Upload do resultado para R2: vinhetas/final/{id}_{ts}.mp3
7. UPDATE vinhetas SET audio_url = <url>, duracao_sec = <medida>
Comando ffmpeg (caso com bed + ducking + intro + outro)
ffmpeg -y \
  -i intro.mp3 \
  -i voice.mp3 \
  -i bed.mp3 \
  -i outro.mp3 \
  -filter_complex "
    [2:a]aloop=loop=-1:size=2e+09,volume={BED_VOLUME_DB}dB[bedloop];
    [1:a][bedloop]sidechaincompress=threshold=0.05:ratio=8:attack=5:release=300[voiced];
    [voiced]apad=pad_dur=0.3[voicedp];
    [0:a][voicedp][3:a]concat=n=3:v=0:a=1[mixed];
    [mixed]loudnorm=I=-16:LRA=11:TP=-1.5[out]
  " \
  -map "[out]" -c:a libmp3lame -b:a 192k -ar 44100 output.mp3
Casos sem bed → remove [2:a]…[bedloop] e troca sidechaincompress por passthrough do [1:a]. Casos sem intro/outro → remove as entradas correspondentes e ajusta concat=n=. Se ducking_enabled = false, troca sidechaincompress por amix=inputs=2:duration=first:weights=1 0.4.

Implemente como um builder que monta o filter_complex dinamicamente conforme os assets disponíveis.

4. Seed das 54 vinhetas — atualizar seed
Ao criar cada vinheta no POST /api/vinhetas/seed, associar:

background_track_id: pegar a primeira background_track cujo tags contenha o nome do bloco (madrugada, prime, etc). Se não houver match, deixar NULL — o pipeline funciona sem bed.
sfx_intro_url / sfx_outro_url: se já existir SFX seedado para aquele tipo (consultar mapa do vinhetas-sfx.service), preencher. Caso contrário, deixar NULL — o pipeline busca no catálogo na hora do gerarAudio.
5. Endpoint de reprocessamento em lote
POST /api/vinhetas/regenerar-todas
  → body: { only_missing_audio?: boolean }   (default false)
  → response: { queued: number }
Roda gerarAudio em sequência (com Promise controlado, max 3 paralelos) em todas as vinhetas ativas. Útil para aplicar a nova mixagem em vinhetas que já existem com áudio antigo.
